# 基面系统评估公式规范

- 公式版本：`research-demo-v1`
- 适用范围：高强度城市片区方案前期比选与作品集 Demo 校核
- 基本原则：保留原始指标单位；仅在综合指标中按显式基准归一化；所有研究假设均可追溯、可替换。

## A. 基面容量评估

### 参数定义

| 参数 | 含义 | 单位 | 建议范围 | 数据来源 |
|---|---|---:|---:|---|
| `A_ground` | 地表层公共空间面积 | ㎡ | ≥0 | CAD/GIS 面域统计、设计总图 |
| `A_underground` | 地下层公共空间面积 | ㎡ | ≥0 | 地下空间总图、BIM/CAD |
| `A_upper` | 空中/地上层公共空间面积 | ㎡ | ≥0 | 连廊、平台、屋顶平面统计 |
| `R` | 容积率 | 无量纲 | >0，常见 0-15 | 规划条件、技术经济指标 |
| `Ni` | 各层有效系数 | 无量纲 | 0-1 | 开放时长、可达性、使用效率标定 |
| `N_overall` | 总体有效系数 | 无量纲 | 0-1 | 研究假设或运营数据标定 |
| `Wi` | 三层理想权重 | 无量纲 | 各项 0-1，合计 1 | 课题目标、专家赋权、案例样本 |
| `A_reference` | 容量计算参考面积，当前采用建设用地面积 | ㎡ | >0 | 技术经济指标 |
| `Pi` | 各层实际面积占比 | 无量纲 | 0-1，合计 1 | 由三层面积计算 |
| `D` | 实际分布与理想权重的接近程度 | 无量纲 | 0-1 | 由 `Pi`、`Wi` 计算 |
| `A_effective` | 有效公共空间面积 | ㎡ | ≥0 | 由 `Ai`、`Ni` 计算 |
| `C` | 基面容量指数 | 无量纲 | ≥0 | 由有效面积比、R、D 计算 |

### 公式

```text
Pi = Ai / ΣAi
D = 1 - 0.5 × Σ|Pi - Wi|, D ∈ [0, 1]
A_effective = Σ(Ai × Ni) × N_overall
C = (A_effective / A_reference) × R × D
```

`A_effective / A_reference` 消除面积量纲，`R` 与 `D` 均为无量纲，因此 `C` 无量纲。D 采用总变差距离的补数，实际分布与理想权重完全一致时 D=1。当前可校核的是计算结构；`Wi`、`Ni` 和参考面积口径仍需真实课题标定。

## B. 连接效率评估

### 参数定义

| 参数 | 含义 | 原始单位 | 无量纲处理/建议范围 | 数据来源 |
|---|---|---:|---|---|
| `P` | 立体基面投影面积 | ㎡ | 与 A 相除得到 MPR | GIS/CAD 投影面 |
| `A` | 影响域面积 | ㎡ | >0 | 步行影响域、研究边界 |
| `N` | 连接节点数 | 个 | 与 S 换算为节点密度 | 路网/竖向交通节点清单 |
| `S` | 立体基面总面积 | ㎡ | >0 | 三层基面面积合计 |
| `MLI` | 层阶度 | 无量纲 | 以 3 为 v1 归一化基准 | 标高与层级关系统计 |
| `Fi` | 连通楼层数 | 层 | 与系统数相除得到 MSCR | 建筑/连廊连通表 |
| `continuousSystems` | 连续基面系统数 | 个 | >0 | 连续公共空间系统识别 |
| `Wf` | 连廊宽度效能因子 | 无量纲 | 建议 0-1.5 | 有效净宽与目标净宽比 |
| `MPR` | 立体基面覆盖率 | 无量纲 | HCI 中以 0.6 为 v1 基准归一化 | `P/A` |
| `MCI` | 连接节点密度 | **个/万㎡** | HCI 中以 1.5 个/万㎡为 v1 基准归一化 | `N/(S/10000)` |
| `HCI` | 水平连接指数 | 无量纲 | 0-1 | MPR、MCI 综合 |
| `MSCR` | 每个连续系统的连通楼层数 | 层/系统 | VCI 中以 8 层/系统为 v1 基准归一化 | `Fi/continuousSystems` |
| `VCI` | 垂直连接指数 | 无量纲评分 | 0-10；进入 TLEI 前除以 10 | MLI、MSCR 综合 |
| `TLEI` | 立体链接效能指数 | 无量纲评分 | 0-3 展示尺度 | HCI、归一化 VCI、Wf 综合 |

### 公式

```text
MPR = P / A
MCI = N / (S / 10000)                       [个/万㎡]
HCI = 0.6×min(MPR/0.6, 1) + 0.4×min(MCI/1.5, 1)

MSCR = Fi / continuousSystems               [层/系统]
VCI_norm = 0.5×min(MLI/3, 1) + 0.5×min(MSCR/8, 1)
VCI = 10 × VCI_norm

TLEI = 3 × min((0.5×HCI + 0.5×VCI_norm) × Wf, 1)
```

MPR、MCI、MSCR 保留原始研究单位。HCI、VCI、TLEI 只使用归一化后的分量，避免 VCI 的 0-10 展示尺度或 MCI 的节点密度量级支配结果。0.6、1.5、3、8 与综合权重是 `research-demo-v1` 校核基准，后续应通过案例分布、规范阈值或专家标定替换。

## C. 设施布局评估

设施模块第一版保留为 Demo 规则评分：比较地下、地表、地上三层的功能性、体验性、社会性、自发性设施比例与目标结构，以各层适配度和跨层均衡度生成 FLI，并用规则权重生成 FE、LA。

该模块不是实证研究公式。后续应替换或扩展为：

- 基于步行网络的设施服务半径与覆盖人口；
- 轨道站点、街道和跨层节点的网络可达性；
- 业态类型、面积、开放时段与目标人群匹配；
- POI、客流热度、停留时间和运营数据校核。

## 版本与替换规则

1. 修改公式时更新 `FORMULA_VERSION`，并在本文件记录归一化基准和权重来源。
2. 保持函数返回的 `result`、`explanation`、`errors` 和 `formulaVersion` 字段，避免界面与报告失去可追溯性。
3. 新公式上线前运行 `tests/formula-check.js`，检查四方案、参数敏感性和异常输入。
4. 页面计算过程会对比 `schemes.json` 的原始展示值；相对偏差超过 20% 时显示 warning，应先确认口径变化再接受新结果。
